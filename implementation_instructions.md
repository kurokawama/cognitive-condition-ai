- **欲求主導型**（恐怖訴求ではない）

### 離脱防止
- スキップパス: 1回/月（連続記録途切れ防止）

### AI表現ルール（PMDA準拠）
- ❌ 「認知症リスクが高い」→ ✅ 「今日は注意力が回復傾向にあります」
- ❌ 「MCIの可能性がある」→ ✅ 「反応時間が少し長めでした。睡眠は十分でしたか？」
- 3原則: 断言しない / 他者比較しない / 行動を提案する
- AIキャラクター名: Phase 1で名前設定（トーン: 丁寧な友人）、ビジュアルはPhase 2

## DB設計方針（Supabase）

### テーブル構成（最小5テーブル）
1. `users` — プロファイル・課金ステータス・**組織ID**（Phase 1からカラム追加）
2. `check_sessions` — チェック実施記録・スコア3種・総合スコア・created_at
3. `ai_conversations` — AIトーク履歴・user_id外部キー
4. `daily_notes` — ひとこと記録（睡眠・気分・忙しさ）
5. `organizations` — 法人向け（Phase 1ではテーブルのみ、UIなし）

### RLS方針
- `check_sessions`: user_idによる行レベル分離（他ユーザーのスコアへアクセス不可）
- `ai_conversations`: 有料プランユーザーのみREAD可能（課金ステータス連動）
- 認知機能データは高機密 → 将来の個人情報保護法対応を念頭に設計

## UI設計原則

### 全般
- **1画面1メッセージ** — 情報密度を最小化
- フォントサイズ最低17px（Dynamic Type対応）
- タップターゲット最低44×44pt（Apple HIG準拠）
- コントラスト比WCAG AA 4.5:1以上

### チェック画面
- テスト開始ボタンは1つだけ（選択肢を与えない）
- テスト中タイマー非表示（焦り防止）
- テスト切替アニメーション0.2秒以内
- 「やり直し」ボタン無効化（迷い防止）

### スコア表示
- 下降時: 赤色・下矢印禁止 → 「変化」として中立表現
- 完了時: 必ず「できました」の肯定フィードバック
- 表示: 数値 + 前日比 + ひとこと解釈の3点セット

## 技術スタック

- **Frontend**: Next.js 15 + TypeScript + Tailwind CSS + shadcn/ui
- **Backend**: Supabase (Auth + Database + RLS + Edge Functions)
- **AI**: 短く安価なモデル（Claude Haiku or GPT-4o-mini）
- **決済**: Stripe（Phase 1はモック、Phase 2で本番接続）
- **通知**: Expo Push Notifications（PWA or React Native）

## 市場ポジション

- 認知症関連市場: 55億円(2024) → 270億円(2030)、**5倍成長**
- 競合との差別化: 「脳トレ」ではなく「コンディション管理」、1分完結、日本語特化
- 最大の脅威: 脳にいいアプリ（完全無料、25+自治体採用）→ 有料AIの品質で差別化
- B2B機会: 企業健康経営パッケージ（法人契約LTV = 個人の3-5倍）
