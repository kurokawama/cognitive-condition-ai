## コンセプト

**"頭の体重計"** — 毎日1分の認知チェックとAIコーチング
- ゲームで鍛えるのではなく「測る・少しハマる・理解する・整える」
- PMDA規制準拠: コンディション表示に限定（診断・リスク予測は禁止）
- **B2Cオンリー**: 個人プロジェクトとして小さく始め、実績ができたらDr.Stretch/Wecleに持ち込む
- **SEO重視**: ウェブアプリとしてSEOで集客する戦略
- **差別化軸**: 「鍛える」ではなく「測る・整える」。1分完結。日本市場特化。体×頭の統合コンディション管理。

### AI表現ルール（PMDA準拠）
- NG: 「認知症リスクが高い」→ OK: 「今日は注意力が回復傾向にあります」
- NG: 「MCIの可能性がある」→ OK: 「反応時間が少し長めでした。睡眠は十分でしたか？」
- 3原則: 断言しない / 他者比較しない / 行動を提案する
- NGワード辞書: 診断, リスク, MCI, 認知症, 可能性, 危険, 悪化 → プロンプトにハードコード

### 通知トーン設計
- 朝リマインド: 「おはようございます。今日のコンディションチェック、準備できています」
- 連続達成: 「7日連続です。ベースラインが安定してきました」
- 復帰促し: 「お疲れさまです。また始めたくなったらいつでもどうぞ」
- **NG**: 催促・監視感のある表現、毎日2通以上、深夜早朝

### 離脱防止メカニズム
- 連続記録カウンター + スキップパス1回/月
- 週次AI分析の自動配信（有料）
- 月次サマリー
- データ蓄積の可視化（スイッチングコスト）
- 「サービスには飽きるが、自分のデータには執着する」原則

## SEO戦略

### 技術SEO（Next.js App Router）
- generateMetadata: 全ルートにMetadata API
- metadataBase: ルートlayoutに設定 → canonical URL自動生成
- JSON-LD構造化データ: FAQ, Article, WebApplication スキーマ
- Dynamic OG画像: 1200x630px、ルートごとに動的生成
- sitemap.ts + robots.ts: canonical URLのみ
- セマンティックHTML: article, section, header, nav

### Core Web Vitals目標
| メトリクス | 目標 | 対策 |
|-----------|------|------|
| LCP | ≤2.5s | Server Components、Image最適化 |
| CLS | ≤0.1 | フォント事前読み込み、画像サイズ指定 |
| INP | ≤200ms | クライアントJS最小化 |
